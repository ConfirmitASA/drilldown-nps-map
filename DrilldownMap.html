colorFn function

A function that allows custom color coding computation based on value and target. colorFn accepts two attributes: value and target and must return a color string based on those two attributes. Make sure hierarchy has target loaded from DBDesigner table into each hierarchy level, otherwise a default config dataClasses takes precedence on value Example:

   {
   //some constructor configuration above
     colorFn: function(value,target){
       return (value!=null && target!=null)? (value - target >= 0) ? "#18BC9C" : ((value >= 0.9*target) ? "#FF4900" : "#E45335") : undefined;
     }
    //some constructor configuration below
   }
valueColumn Number <optional>
 1

Zero-based column index that contains primary value which will be used for map coloring
